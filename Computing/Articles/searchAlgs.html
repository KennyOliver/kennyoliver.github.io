But how does it all work?

Glad you asked!
There are 2 main algorithms: linear & binary.

They are used to find data (in this case, websites) within a dataset (a list).


Linear

Linear search works by searching a dataset, and checking whether the requested data matches the . If there isn't a match, the next item in succession will be checked.

Binary

Binary is based upon the "Divide & Conquer" plan.

Binary search works by comparing the requested data value to the middle item in the list.
It will check whether the middle value matches, is higher, or is lower.
Then, if the middle item doesn't match, it will cut off the items that are either higher or lower than that middle value, so that there is less to search through.
The algorithm will keep reiterating until an item matches the requested data value, and then return it.

In conclusion, it takes the dataset and keeps splitting it in half until the item is found.

Just to sum up...

You'll need both, each for different scenarios.

Linear is great if you haven't organised the dataset before it's processed,
however it can take a long time to process the entire dataset.

Binary is great if you need the dataset to be processed quickly,
however it must be sorted before it's processed.

You just have to pick whatever matches your needs.
